# 项目结构说明

## 用途与技术栈

本项目是 AI Agent Skills 周报静态网站：Python 从 GitHub API 采集数据，保存原始数据和每周快照，处理脚本计算分类、评分与 Star 增长，Vite + React 构建可部署的静态页面。目标部署平台是 GitHub Pages。

## 关键数据流

`fetch_skills.py` → `data/raw/latest.json` 与 `data/snapshots/YYYY-MM-DD.json` → `process_skills.py` → `data/processed/skills.json` → Vite → `dist/`。

## 目录职责

- `scripts/`: GitHub 采集、README 摘要、数据处理和 SEO 文件生成。
- `data/raw/`: 最新一次原始采集结果。
- `data/snapshots/`: 每周历史快照，用于 Star 增长计算。
- `data/processed/`: 前端直接导入的聚合数据。
- `src/`: React 页面、组件、类型和样式。
- `public/`: 静态资源；部署时复制到 `dist/`。
- `dist/`: 构建产物，可直接部署，不作为源码编辑。
- `.github/workflows/`: 首次推送和每周自动更新、构建、Pages 部署。

## 配置与注意事项

采集脚本读取 `GH_TOKEN` 或 `GITHUB_TOKEN` 环境变量，不把令牌写入文件。未配置令牌时也能运行，但 GitHub API 速率限制更严格，结果可能较少。不要提交 `.env`、访问令牌或个人数据。
